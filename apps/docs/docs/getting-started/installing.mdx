# Installing ADTEC Code

ADTEC Code is distributed as an internal VSIX package. Obtain the approved package from your ADTEC Code administrator or your organization's internal software channel.

## Install the Internal VSIX

1. Open VS Code.
2. Open the Extensions view.
3. Open the Extensions view menu and select **Install from VSIX...**.
4. Select the approved ADTEC Code `.vsix` package.
5. Reload VS Code if prompted.

After installation, open the ADTEC Code panel from the Activity Bar.

The command-line equivalent is:

```shell
code --install-extension <path-to-adtec-code.vsix>
```

## ADTEC Code Nightly

Nightly packages, when available internally, follow the same VSIX installation process. See [ADTEC Code Nightly](/advanced-usage/adtec-code-nightly).

## Troubleshooting

**Extension Not Visible**

- Restart VS Code.
- Verify ADTEC Code is listed and enabled in Extensions.
- Try disabling and re-enabling the extension.
- Check the Output panel for errors: select **View > Output**, then select `ADTEC-Code`.

**Installation Problems**

- Verify VS Code version 1.84.0 or later.
- Confirm the VSIX package came from the approved internal distribution channel.
- Ask your ADTEC Code administrator for a package compatible with your editor and platform.

## Getting Support

Contact your ADTEC Code administrator and include the `ADTEC-Code` output-channel logs.
