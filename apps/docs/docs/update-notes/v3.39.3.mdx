# ADTEC Code 3.39.3 Release Notes (2026-01-10)

This release renames ADTEC Code Cloud Provider to ADTEC Code Router and improves CLI and internal organization.

<img src="/img/v3.39.3/v3.39.3.png" alt="ADTEC Code v3.39.3 Release" width="600" />

## ADTEC Code Router rename

ADTEC Code now shows **ADTEC Code Router** anywhere the extension previously referred to the **ADTEC Code Cloud Provider**. The new name is meant to make the intent clearer: ADTEC Code Router is a router-style provider (similar in idea to other router/aggregator services), but with a **curated set of models we’ve tested with ADTEC Code**. ([#10560](https://github.com/Y-vQv-Y)).

## QOL Improvements

* **ADTEC Code Router naming consistency**: Updates the ADTEC Code Router service name and link so the UI points at the correct destination ([#10607](https://github.com/Y-vQv-Y))
* **ADTEC Code Router naming consistency**: Updates router naming in shared types so labels stay consistent across the UI ([#10605](https://github.com/Y-vQv-Y))
* **ExtensionHost organization**: Improves internal organization around mode changes and related events to help keep behavior consistent ([#10600](https://github.com/Y-vQv-Y))
* **CLI release testing**: Adds options to build and install the CLI release locally for testing ([#10597](https://github.com/Y-vQv-Y))
* **CLI organization**: Reorganizes the CLI file structure to make future changes easier to maintain ([#10599](https://github.com/Y-vQv-Y))

## Misc Improvements

* **CLI TUI**: Adds an interactive terminal UI mode to the ADTEC Code CLI ([#10480](https://github.com/Y-vQv-Y))
