# ADTEC Code 3.36.3 Release Notes (2025-12-10)

This release adds browser screenshot saving, xhigh reasoning effort support, native tools default for OpenRouter, plus provider updates and bug fixes.

<img src="/img/v3.36.3/v3.36.3.png" alt="ADTEC Code v3.36.3 Release" width="600" />

## Browser Screenshot Saving

The browser tool now supports saving screenshots to a specified file path with a new `screenshot` action, enabling you to capture visual state during browser automation tasks. ([#9963](https://github.com/Y-vQv-Y))

## Extra-High Reasoning Effort

Users of the gpt-5.1-codex-max model with the OpenAI provider can now select "Extra High" as a reasoning effort level, enabling maximum reasoning capability for complex tasks (thanks andrewginns!) ([#9900](https://github.com/Y-vQv-Y))

## OpenRouter Native Tools Default

OpenRouter models that support native tools now automatically use native tool calling by default, improving tool calling reliability without manual configuration. ([#9878](https://github.com/Y-vQv-Y))

## QOL Improvements

* **Unified Context-Management UX**: Real-time feedback for context operations with truncation notifications and condensation summaries ([#9795](https://github.com/Y-vQv-Y))
* **Better OpenAI Error Messages**: Enhanced error handler extracts detailed information from API errors for easier troubleshooting ([#9639](https://github.com/Y-vQv-Y))
* **Token Counting Optimization**: Removed separate API calls for token counting, improving performance ([#9884](https://github.com/Y-vQv-Y))
* **Tools Decoupled from System Prompt**: Tool-specific instructions are now self-contained in tool descriptions ([#9784](https://github.com/Y-vQv-Y))

## Bug Fixes

* **Tool Protocol Selector**: Always show tool protocol selector for OpenAI-compatible providers (thanks bozoweed!) ([#9966](https://github.com/Y-vQv-Y))
* **apply_diff Filtering**: Properly exclude apply_diff from native tools when diff is disabled (thanks denis-kudelin!) ([#9920](https://github.com/Y-vQv-Y))
* **API Timeout Handling**: Fixed disabled API timeout (set to 0) causing immediate request failures (thanks dcbartlett!) ([#9960](https://github.com/Y-vQv-Y))
* **Reasoning Effort Dropdown**: Respect explicit supportsReasoningEffort array values and fix disable handling ([#9970](https://github.com/Y-vQv-Y), [#9930](https://github.com/Y-vQv-Y))
* **Actual Error Messages**: Display actual API error messages instead of generic text on retry ([#9954](https://github.com/Y-vQv-Y))
* **Stream Hanging Fix**: Process finish_reason to emit tool_call_end events for OpenRouter, ADTEC Code, and xAI providers ([#9927](https://github.com/Y-vQv-Y), [#9929](https://github.com/Y-vQv-Y))
* **tool_result ID Validation**: Validate and fix tool_result IDs before API requests to prevent Anthropic rejections ([#9952](https://github.com/Y-vQv-Y))
* **Suppressed Internal Error**: Fixed 'ask promise was ignored' error leaking to LLM conversations ([#9914](https://github.com/Y-vQv-Y))
* **Provider Sanitization**: Fixed infinite loop when using removed/invalid API providers ([#9869](https://github.com/Y-vQv-Y))
* **Context Icons Theme**: Context-management icons now use foreground color to match VSCode themes ([#9912](https://github.com/Y-vQv-Y))
* **Eval Runs Deletion**: Fixed foreign key constraint preventing eval run deletions ([#9909](https://github.com/Y-vQv-Y))
* **OpenAI-Compatible Timeout Reliability**: Added timeout handling to OpenAI-compatible providers to prevent indefinite hangs (thanks dcbartlett!) ([#9898](https://github.com/Y-vQv-Y))
