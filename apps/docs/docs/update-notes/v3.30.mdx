# ADTEC Code 3.30 Release Notes (2025-11-03)

This release adds OpenRouter embeddings and Bedrock Global Inference, updates provider defaults and limits, and delivers stability and UI improvements.

<img src="/img/v3.30.0/v3.30.0.png" alt="ADTEC Code v3.30 Release" width="600" />

## OpenRouter Embeddings

We've added OpenRouter as an embedding provider for codebase indexing in ADTEC Code (thanks dmarkey!) ([#8973](https://github.com/Y-vQv-Y)).

OpenRouter currently supports 7 embedding models, including the top‑ranking Qwen3 Embedding.

> **📚 Documentation**: See [Codebase Indexing](/features/codebase-indexing) and [OpenRouter Provider](/providers/openrouter).

## QOL Improvements

* Terminal settings cleanup with Inline as the default terminal and clearer options; shell integration default is disabled to reduce environment conflicts ([#8342](https://github.com/Y-vQv-Y))
* File path tooltips show full paths on hover across reads/edits, directory listings, and search, reducing confusion in narrow layouts (thanks da2ce7!) ([#9030](https://github.com/Y-vQv-Y))
* Home screen visuals refined for clarity and space usage ([#9057](https://github.com/Y-vQv-Y))
* Chat diff appearance is clearer and easier to scan ([#8932](https://github.com/Y-vQv-Y))
* Settings copy clarifies that an Error & Repetition Limit of 0 disables the mechanism ([#8965](https://github.com/Y-vQv-Y))

## Bug Fixes

* Prevent message loss during queue drain race conditions to preserve message order and reliable chats ([#8955](https://github.com/Y-vQv-Y))
* Requesty OAuth: auto-create a stable "Requesty" profile with a default model so sign-in completes reliably (thanks Thibault00!) ([#8699](https://github.com/Y-vQv-Y))
* Cancel during streaming no longer causes flicker; you can resume in place, input stays enabled, and the spinner stops deterministically ([#8986](https://github.com/Y-vQv-Y))
* Remove newline-only reasoning blocks from OpenAI-compatible responses for cleaner output and logs ([#8990](https://github.com/Y-vQv-Y))
* "Disable Terminal Shell Integration" now links to the correct documentation section ([#8997](https://github.com/Y-vQv-Y))
* Correct OpenRouter Mistral embedding dimension to 1536 to prevent vector size errors; existing Qdrant collections are recreated with the correct size for consistent similarity search ([#9028](https://github.com/Y-vQv-Y))
* Revert the recent cancel/resume change that caused UI flicker and unreliable resumption, restoring the previous stable behavior ([#9032](https://github.com/Y-vQv-Y))
* Eliminate UI flicker during task cancellation; the task view remains stable ([#9037](https://github.com/Y-vQv-Y))
* Keep pinned models fixed at the top of long lists for faster access (thanks XiaoYingYo!) ([#8813](https://github.com/Y-vQv-Y))
* Ensure Checkpoints commit to ADTEC Code’s shadow repository even when GIT_DIR is set in Dev Containers (thanks nonsleepr!) ([#8811](https://github.com/Y-vQv-Y))
* Restore correct 32K maxTokens for Claude Opus 4.1 to avoid premature truncation (thanks kaveh-deriv!) ([#9046](https://github.com/Y-vQv-Y))
* Fix dynamic provider model validation so provider switches use a valid default model without TypeErrors (thanks NotADev137!) ([#9054](https://github.com/Y-vQv-Y))
* Include full user agent details in AWS Bedrock requests for better auditing and troubleshooting (thanks ajjuaire!) ([#9043](https://github.com/Y-vQv-Y))
* Auto‑retry on empty assistant response to prevent task cancellations (reporter: Akillatech) ([#9083](https://github.com/Y-vQv-Y))

* Prevent notification sound when messages are queued to auto‑continue ([#8540](https://github.com/Y-vQv-Y))

## Misc Improvements
