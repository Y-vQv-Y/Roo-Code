# ADTEC Code 3.29.0 Release Notes (2025-10-24)

This release adds intelligent file reading, enables browser-use for all image-capable models, and adds dynamic ADTEC Code Cloud model loading — plus many fixes and model updates.

<img src="/img/v3.29.0/v3.29.0.png" alt="ADTEC Code v3.29.0 Release" width="600" />

## Introducing ADTEC Code's first Cloud Agent, the PR Reviewer

It runs ADTEC Code in the cloud, giving extremely high quality code reviews instantly. We’ve been using it heavily to build ADTEC Code and now it's also available to the community.

Learn more: https://github.com/Y-vQv-Yreviewer

## QOL Improvements

* Intelligent file reading with token‑budget management and a 100KB preview for very large files (thanks daniel-lxs, liwilliam2021!) ([#8789](https://github.com/Y-vQv-Y))
* Browser‑use enabled for all image‑capable models (thanks app/ADTEC, hannesrudolph!) ([#8121](https://github.com/Y-vQv-Y), issue #8116)
* Reduce ui_messages.json bloat by removing GPT‑5 instructions/reasoning_summary ([#8756](https://github.com/Y-vQv-Y))
* Adjustable checkpoint initialization timeout and clearer warnings (thanks NaccOll!) ([#8019](https://github.com/Y-vQv-Y), issue #7843)
* Improve auto‑approve button responsiveness ([#8798](https://github.com/Y-vQv-Y))
* Retry API requests on stream failures instead of aborting the task ([#8794](https://github.com/Y-vQv-Y))
* Improve checkpoint menu translations ([#8796](https://github.com/Y-vQv-Y))
* Try a 5s status mutation timeout to reduce flaky status changes ([#8734](https://github.com/Y-vQv-Y))

## Bug Fixes

* apply_diff export preserves trailing newlines (fix stripLineNumbers) ([#8227](https://github.com/Y-vQv-Y), issue #8020)
