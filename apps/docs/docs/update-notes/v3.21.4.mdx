# ADTEC Code 3.21.4 Release Notes (2025-06-23)

This release introduces the new Claude Code provider for local CLI integration and fixes several critical issues.

## Use Your Claude Max Subscription with ADTEC Code

You can now use your Claude Max subscription directly in ADTEC Code through our new Claude Code provider (thanks ADTEC Code!) ([#4864](https://github.com/Y-vQv-Y), [#5038](https://github.com/Y-vQv-Y)):

- **Leverage Your Existing Subscription**: Connect your Claude Max plan to ADTEC Code through the Claude CLI
- **No Additional API Costs**: Use your subscription benefits instead of paying per-token API rates
- **Access Premium Models**: Use Claude Sonnet 4, Opus 4, and other advanced models included in your plan
- **Zero Setup Complexity**: Select Claude Code as your provider during initial setup - no API keys needed
- **Advanced Reasoning Support**: Full access to Claude's thinking modes and reasoning capabilities

Perfect for Claude Max subscribers who want to maximize their subscription value while coding.

## Bug Fixes

* **Apply Diff Accuracy**: Fixed start line parameter not working correctly when applying multiple file diffs (thanks samhvw8!) ([#5019](https://github.com/Y-vQv-Y))
* **Ollama Compatibility**: Resolved validation errors that prevented Ollama from working with certain models (thanks daniel-lxs, MartinHarding1998!) ([#5014](https://github.com/Y-vQv-Y))
